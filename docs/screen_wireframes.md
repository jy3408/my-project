## 2. 인터랙션 원칙

> 모든 애니메이션은 **100~400ms** 범위에서 처리한다. 느리면 답답하고, 빠르면 어색하다.

### 2-1. 전역 인터랙션 규칙

| 상황 | 애니메이션 | 지속 시간 | 이징 |
|------|----------|----------|------|
| 화면 전환 (앞으로) | 오른쪽에서 슬라이드인 | 280ms | ease-out cubic |
| 화면 전환 (뒤로) | 왼쪽으로 슬라이드아웃 | 280ms | ease-in cubic |
| 카드/리스트 최초 등장 | 아래에서 페이드업 | 400ms | ease-out cubic |
| 버튼 탭 | 살짝 눌리는 효과 (scale 0.97) | 100ms | ease |
| 모달 등장 | 아래에서 슬라이드업 | 320ms | spring |

### 2-2. 화면별 핵심 인터랙션

#### 화면 1: 스플래시
- **로고 등장**: 앱 실행 시 로고가 아래에서 페이드업 (delay 200ms)
- **슬로건 등장**: 로고 이후 stagger로 한 줄씩 페이드인 (delay 50ms씩)
- **CTA 버튼 등장**: 전체 텍스트 이후 0.6초 뒤 페이드인
- **배경**: 연한 그린 계열 그라데이션 메시 정적 배경

#### 화면 2: 회원가입
- **인풋 포커스**: 테두리 색상 Slate 300 → Emerald 500 전환 (200ms)
- **비밀번호 유효성 체크**: 조건 충족 시 회색 텍스트 → 초록색 + 체크 아이콘으로 전환 (150ms)
- **비밀번호 불일치**: 두 번째 인풋 테두리 Red 500으로 전환 + 좌우 흔들림(shake) 애니메이션 3회
- **다음 버튼 활성화**: 모든 조건 충족 전 버튼 비활성(불투명도 40%) → 충족 시 활성화 페이드 (200ms)

#### 화면 3 & 4: 질환/알레르기 선택
- **칩 선택**: 탭 시 배경 White → Emerald 600, 텍스트 Slate 800 → White, 테두리 제거 (150ms)
- **칩 해제**: 역방향 동일 전환
- **"없음" 칩 선택**: 다른 선택된 칩들이 동시에 해제되며 살짝 흔들리는 효과
- **진행바**: 화면 진입 시 현재 단계까지 너비가 왼쪽에서 채워짐 (400ms, ease-out)
- **다음 버튼**: 아무것도 선택 안 했을 때 탭 → 버튼 shake + "하나 이상 선택해 주세요" 토스트

#### 화면 5: 메인 / 냉장고 스캔
- **카메라 스캔 버튼**: 탭 시 카메라 아이콘이 0.5초간 셔터 깜빡임 효과
- **스캔 로딩**: 분석 중 원형 스피너 + "AI가 식재료를 인식 중입니다..." 텍스트
- **최근 식재료 골든타임 배지**: 주황색 배지에 pulse-glow 애니메이션 상시 적용
- **하단 탭 전환**: 활성 탭 아이콘 scale 1 → 1.15 + 색상 변경 (150ms)

#### 화면 6: 스캔 결과 ⭐ 핵심 인터랙션
- **카드 등장**: 위에서 아래로 50ms 간격 stagger fadeUp (총 8장 = 약 400ms 내 완료)
- **골든타임 배지 pulse-glow**:
  ```
  0%   → box-shadow: 0 0 0 0px rgba(249,115,22, 0.6)
  50%  → box-shadow: 0 0 0 6px rgba(249,115,22, 0.0)
  100% → box-shadow: 0 0 0 0px rgba(249,115,22, 0.0)
  반복 주기: 2s infinite
  ```
- **기한초과 배지**: 연속 pulse 없음. 대신 카드 진입 시 1회 빨간 테두리 flash (300ms)
- **차단 카드**: 카드 배경이 연한 Red 50으로 tinted, 차단 아이콘 등장 시 scale 0 → 1 bounce (250ms)
- **대체재 칩**: 차단 카드 등장 0.2초 후 아래에서 슬라이드업 (200ms)
- **수량 배지**: 카드 이름 아래 살짝 늦게(delay 100ms) 페이드인
- **요약 바 숫자**: 화면 진입 시 0에서 실제 숫자까지 카운트업 (600ms)

#### 화면 7: 레시피 추천
- **골든타임 재료 하이라이트**: 상단 바 배경 주황 그라데이션 + 재료 칩 좌우로 천천히 흔들리는 shimmer
- **레시피 카드 등장**: 스크롤 시 뷰포트 진입 감지 → fadeUp 트리거 (IntersectionObserver)
- **환경 임팩트 숫자**: 카드가 화면에 들어오는 순간 0 → 실제 값 카운트업 (1400ms, ease-out cubic)
- **진행 바**: 카운트업과 동시에 왼쪽에서 오른쪽으로 채워짐
- **"부족 재료 구매" 버튼**: 탭 시 외부 앱으로 딥링크 이동 전 0.3초 로딩 스피너

### 2-3. 에러 & 빈 상태

| 상황 | 처리 방법 |
|------|----------|
| 스캔 실패 (이미지 인식 불가) | 중앙 모달: "인식에 실패했습니다. 더 밝은 환경에서 다시 시도해 주세요" + 재시도 버튼 |
| 인식된 식재료 0개 | 빈 상태 일러스트 + "식재료를 찾지 못했어요. 수동으로 추가해 볼까요?" |
| 네트워크 오류 | 상단에서 내려오는 토스트 배너 (Red 배경, 2초 후 자동 사라짐) |
| 모든 재료가 차단됨 | "사용 가능한 재료가 없어요" 안내 + 대체재 바로가기 버튼 강조 |

### 화면 1: 스플래시 / 온보딩
**대응 흐름:** PRD Step 1 (앱 최초 진입)
**배경:** Emerald 800 → Emerald 600 수직 그라데이션

```
┌─────────────────────────────────┐
│                                 │
│  (배경: 그린 그라데이션)        │
│                                 │
│        🥬 NutriLens             │  ← 폰트 28px / Bold 800
│                                 │
│   건강을 담은 스마트 냉장고     │  ← 16px / Regular
│   내 몸에 맞는 식재료만 골라,  │
│   가장 먼저 먹어야 할 것을      │
│   알려드려요                    │
│                                 │
│                                 │
│  ╔═════════════════════════╗   │
│  ║    시작하기  →          ║   │  ← Primary 버튼 / 높이 52px
│  ╚═════════════════════════╝   │
│                                 │
│   이미 계정이 있으신가요?       │
│   ─────── [ 로그인 ] ───────   │  ← Ghost 버튼
│                                 │
└─────────────────────────────────┘
```

**인터랙션 요약**
- 로고 → 슬로건 → CTA 순으로 stagger 페이드업 등장
- "시작하기" 탭: scale 0.97 → 1 (100ms) + 다음 화면으로 슬라이드 전환

│                                 │
│   비밀번호                      │
│  ┌───────────────────────────┐  │
│  │  ••••••••            👁   │  │  ← 눈 아이콘: 비밀번호 토글

│  ┌───────────────────────────┐  │

│                                 │
│  ─────────── 또는 ───────────  │

**인터랙션 요약**

- 비밀번호 불일치 시 shake 애니메이션 + 빨간 테두리
- 모든 조건 충족 전까지 "다음 단계" 버튼 비활성 유지
