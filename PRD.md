# 뉴트리렌즈 (NutriLens) - 상세 기획 문서 (PRD)

---

## 1. 문서 개요

- **프로젝트명:** 뉴트리렌즈 (NutriLens)
- **미션:** AI 시각 지능을 활용하여 개인의 건강 상태에 최적화된 식재료 관리 및 레시피 솔루션 제공.

---

## 2. 타겟 사용자 및 문제점

**페르소나 1 (질환자/관리군)**
당뇨, 알레르기 등으로 특정 성분을 엄격히 배제해야 하지만 메뉴 선정에 어려움을 겪는 사용자.

**페르소나 2 (1인 가구/직장인)**
식재료를 대량 구매하지만 유통기한 내 소진하지 못해 버리는 일이 잦은 사용자.

---

## 3. 핵심 기능

- **AI 이미지 스캐너:** 카메라 촬영 한 번으로 냉장고 속 식재료 종류 및 상태 자동 인식.
- **건강 데이터 연동:** 사용자의 건강 프로필(당뇨 여부, 알레르기 정보) 연동 및 필터링.
- **맞춤형 레시피 추천:** '보유 재료' + '건강 상태'를 조합한 맞춤형 레시피 제안.
- **유통기한 알림:** 식재료별 유통기한 추적 및 소진 임박 알림 서비스.

---

## 4. 사용자 흐름

### Step 1. 가입 및 건강 프로필 구축
- 이메일 인증을 통해 보안 계정을 생성한다.
- 서비스의 핵심 로직인 개인별 만성 질환(당뇨 등) 및 알레르기 정보를 정밀 설정한다.
- > **핵심:** 이 단계에서 설정된 데이터는 이후 모든 레시피 추천의 **필터링 기준**이 된다.

### Step 2. 식재료 데이터 입력
- 냉장고 내부 사진을 촬영하거나 업로드한다.

### Step 3. AI 정밀 분석
- AI 시각 지능이 사진 속 식재료의 종류 및 상태를 자동으로 인식한다.
- 사용자 건강 프로필과 대조하여 **금지 재료를 필터링**한다.
- **유통기한 및 신선도 분석**을 동시에 수행한다.

### Step 4. 골든타임 재료 선별
- 유통기한이 가장 임박한 식재료를 **최우선 활용 재료(골든타임 재료)**로 확정한다.

### Step 5. 개인 맞춤형 솔루션 제안
- **건강 상태** + **골든타임 재료**를 조합하여 안전하고 맛있는 오늘 저녁 메뉴를 추천한다.

### Step 6. 서비스 확장 (이커머스 연동)
- 추천 레시피에 부족한 재료를 자동으로 확인한다.
- 이커머스(쿠팡, 컬리 등) 연동을 통해 부족 재료를 즉시 구매할 수 있는 링크를 제공한다.

---

## 5. 상세 기능 명세

### 5.1 계정 가입 및 온보딩

#### 계정 생성
- **이메일 인증:** 가입 시 인증 메일 발송 → 링크 클릭 확인 후 계정 활성화. 미인증 계정은 로그인 차단.
- **비밀번호 정책:** 8자 이상 20자 이하, 영문·숫자·특수문자 각 1자 이상 포함 필수.

#### 건강 프로필 설정 (필수 2단계)
- **만성 질환 선택:** 당뇨, 고혈압, 고지혈증 등 만성 질환 리스트 제공. 복수 선택 가능. 해당 없는 경우 '없음' 명시적 선택 필수.
  - *선택 옵션:* 건강검진 공공데이터 API 연동을 통해 최근 3년간 건강검진 기록 자동 불러오기 지원.
- **알레르기 설정:** 견과류, 갑각류, 유제품, 밀 등 표준 알레르기 카테고리 제공. 복수 선택 가능. 해당 없는 경우 '없음' 명시적 선택 필수.

> **필수 관문(Gate) 로직:** 만성 질환·알레르기 설정 2단계를 모두 완료하지 않으면 이후 서비스(식재료 입력, 레시피 추천 등) 진입 불가. 이 데이터는 이후 모든 필터링 로직의 기준이 되므로 건너뛰기 허용 불가.

#### 보안
- **건강 데이터 암호화:** 만성 질환·알레르기 등 민감 건강 정보는 DB 저장 시 반드시 암호화(AES-256 이상) 처리. 평문 저장 금지.

---

### 5.2 식재료 데이터 입력

- **카메라 촬영:** 모바일 카메라 권한 허용 후 앱 내 직접 촬영. 권한 미허용 시 기능 비활성화 안내.
- **갤러리·파일 업로드:** 기기 갤러리 또는 파일 탐색기를 통한 이미지 업로드 지원.
- **복수 이미지 지원:** 위 두 방식 모두 1회에 여러 장(n개) 업로드 가능. 냉장고 칸별 분할 촬영 등 다양한 상황 커버.
- **용량 제한 정책:** 서버 부하 방지 및 분석 품질 보장을 위해 이미지 1장당 최대 **10MB**, 1회 업로드 최대 **20장**으로 제한. 초과 시 즉시 안내 메시지 노출.

---

### 5.3 AI 비전 분석 엔진

#### 이미지 업로드 API
- 클라이언트 → 서버로 이미지 전송 시 `multipart/form-data` 형식 사용. 업로드 완료 후 분석 작업을 비동기(async) 큐에 등록하여 응답 지연 최소화.

#### 객체 탐지 (Object Detection)
- Gemini 멀티모달 모델을 통해 이미지에서 **식재료 명칭, 수량** 추출.
- **신선도 추정 이중 로직:**
  1. *시각적 신선도:* 이미지 내 식재료의 색상·형태를 AI가 분석하여 상태 추정 (신선 / 주의 / 폐기 권고).
  2. *유통기한 기반 신선도:* 이미지 내 유통기한 텍스트를 OCR로 추출 → 사진 업로드 날짜와 비교하여 잔여 가용 일수 산출.
  - 두 방식 중 더 보수적인(짧은) 값을 최종 신선도 지표로 채택.

#### 필터링 로직
- 인식된 재료 목록과 사용자 금지 재료(만성 질환·알레르기 기반) 교집합 탐지.
- 금지 재료 발견 즉시 **경고 배지** 노출 및 추천 재료 풀에서 자동 제외.

---

### 5.4 골든타임 알고리즘

- **유통기한 데이터 입력 방식:**
  - 바코드 스캔 자동 입력
  - 수동 직접 입력 (보정 가능)
  - 5.3의 AI 비전 분석에서 추출된 OCR 유통기한 자동 참조
- **우선순위 큐(Priority Queue):** 등록된 모든 식재료를 소비 기한 오름차순으로 정렬. 유통기한이 가장 짧은 재료에 레시피 추천 가중치를 높게 부여하여 식재료 낭비 최소화.

---

### 5.5 레시피 추천 및 커머스 연동

#### 대체 재료 추천 (Smart Substitution)
- 골든타임 재료가 사용자 프로필상 금지 재료인 경우, 해당 재료를 제외하는 것에 그치지 않고 AI가 안전한 **대체 재료를 제안**.
  - 예: 설탕 → 스테비아 (당뇨), 밀가루 → 아몬드 가루 (글루텐 알레르기), 새우 → 두부 (갑각류 알레르기)
- 대체 재료 제안은 LLM 프롬프트에 사용자 건강 프로필과 원재료 영양 성분을 함께 전달하여 영양학적 유사성을 고려한 추천 생성.

#### LLM 기반 레시피 생성
- 입력값: 골든타임 재료 목록 + 건강 프로필(금지 성분) + 대체 재료 후보 + 선호 조리 방식
- Gemini에게 구조화된 프롬프트로 전달 → 재료명, 분량, 조리 순서, 영양 정보 포함한 레시피 JSON 반환.

#### 이커머스 딥링크 생성
- 추천 레시피의 부족 재료를 자동 감지.
- 쿠팡(Coupang), 컬리(Kurly) 파트너 딥링크(Deep Link) 규칙에 따라 해당 재료 검색 URL 자동 생성 및 장바구니 바로가기 제공.

---

## 6. 비즈니스 모델

- **구매 중개 수수료:** 레시피에 필요한 부족 식재료 구매 시 이커머스(쿠팡, 컬리 등) 파트너십 수수료.
- **프리미엄 구독:** 전문 영양사의 밀착 케어 리포트 및 질환 맞춤형 정밀 식단 가이드 제공.
- **데이터 사업:** 비식별 처리된 사용자 식습관 데이터를 분석하여 유통사에 트렌드 리포트 제공.

---

## 7. 기술 스택

> Andrej Karpathy의 'Simplicity First' 철학 기반 — 검증된 기술을 최소한으로, 복잡도 없이.

### 프론트엔드 (Frontend)
- **React Native (하이브리드 구조)** — 네이티브 모듈과 WebView를 역할에 따라 분리 운용.
  - **네이티브 모듈:** 카메라, 센서 등 디바이스 하드웨어 제어가 필요한 기능에 집중.
  - **WebView:** 레시피 콘텐츠, 식재료 목록 등 UI 중심 화면을 웹으로 렌더링. 콘텐츠 업데이트 시 앱 재배포 없이 즉시 반영 가능.

### 백엔드 (Backend)
- **FastAPI (Python)** — Python 생태계를 활용한 Gemini AI 모델 연동 최적화. 비동기(async) 기반 이미지 처리 파이프라인으로 냉장고 사진 분석 지연 최소화.
- **PostgreSQL** — 신뢰성 높은 관계형 데이터베이스. 사용자 건강 프로필 및 레시피 데이터 저장.

### AI 모델
- **Gemini 1.5 Pro** — 멀티모달 기능을 통해 냉장고 사진을 정확히 분석. 텍스트+이미지 동시 처리 가능.

### 인프라 (Infrastructure)
- **Supabase** — PostgreSQL 기반 백엔드 서비스로 인증, DB, 스토리지를 한 곳에서 관리. 초기 개발 속도 극대화.
