### 생성된 파일

| 파일 | 설명 |
|------|------|
| `backend/data/allergens.py` | 식약처 지정 22종 알레르기 마스터 데이터 + 질환별 금지 식재료 매핑 |

| `backend/database.py` | Supabase 클라이언트 초기화 + 테이블 스키마 주석 |
| `backend/main.py` | FastAPI 진입점 — 5개 엔드포인트 정의 |
| `backend/requirements.txt` | Python 의존성 5종 |
| `backend/data/__init__.py` | data/ 패키지 인식용 init 파일 |
| `docs/screen_wireframes.md` | 7개 화면 텍스트 와이어프레임 초안 |
| `.gitignore` | Python + .env 표준 제외 설정 |

### 핵심 기술 결정

**골든타임 점수 공식 (0~100점):**
```
유통기한 없음   → 0점  (무시)
이미 지남       → 100점 (최우선 경고)
3일 이내        → 90 + (3 - days_left) × 3
7일 이내        → 50 + (7 - days_left) × 5
그 외           → max(0, 50 - days_left)
```

**필터링 방식:** 부분 문자열 매칭(partial string matching) — 사용자 건강 프로필 키워드를 식재료명과 대조하여 교집합 탐지.

### 트러블슈팅
- `data/` 폴더를 Python 패키지로 인식하지 못하는 문제 → `__init__.py` 추가로 해결

## 세션 7 — 진짜 모바일 앱 UX 전면 재설계

**날짜:** 2026-03-13
**작업 유형:** UX 구조 전면 개편

### 배경
Chloe: "지금 데모는 그냥 내용만 긴 웹사이트 같아. 진짜 핸드폰 앱 느낌이어야 해."

**구체적 요구사항:**
1. 화면 맨 아래 항상 고정된 하단 네비바 (홈/스캔/레시피/리포트 + 아이콘)
2. 탭 전환 시 스크롤 없이 화면이 즉시 바뀌는 앱 방식
3. 상단에 NutriLens 로고 항상 고정

### 구현 방식

**화면 전환 아키텍처:**
```css
.screen { position: absolute; inset: 0; opacity: 0; pointer-events: none; }
.screen.active { opacity: 1; pointer-events: all; }
/* transition 없음 = 즉시 전환 */
```

**레이아웃 구조:**
```
[ .app-topbar ]  ← 고정 상단 (NutriLens 로고 + 알림/프로필)
[ .screen-wrap ] ← 스크롤 영역 (4개 화면 중 1개만 active)
[ .app-nav ]     ← 고정 하단 (4탭 + SVG 아이콘 + active 인디케이터)
```

### 추가된 화면
- **리포트 화면** (신규): 주간 CSS 바차트 (오늘 하이라이트), 성취 뱃지 4종 (2종 잠금), 환경 카드

### JS 핵심 함수
- `switchTab(tab)` — opacity 토글로 즉시 화면 전환
- `startLoading()` → 4단계 AI 시뮬레이션 → `showResults()`
- `startCarbonAnim()` — count-up 애니메이션 + 바 채우기 (레시피 탭 진입 시 1회 실행)
